# FluAgent 智能体架构与使用方法总结

> 版本：v3.0.0 | 记录时间：2026-04-08

---

## 一、项目概述

**FluAgent** 是一个面向**流感病毒生物信息学分析**的 AI Agent 应用，版本 v3.0.0。核心思路是：由 LLM（大语言模型）通过 **Function Calling** 机制自动调度 37 个生物信息学命令行工具，实现从序列质控到进化分析的全流程自动化。

## 三、Agent 核心机制

### 3.1 主要运行模式

| 模式 | 方法 | 说明 |
|------|------|------|
| **FC 循环模式** | `chat()` | 简单任务：LLM 直接决定工具调用，循环执行直到完成 |
| **三角色模式** | `chat_v2()` | 复杂任务：Ask → Plan → Craft 三阶段串行处理 |
| **流式模式** | `chat_stream()` | 异步 SSE 流式输出，配合前端使用 |

### 3.2 三角色（Ask-Plan-Craft）工作流

```
用户输入
  ↓
[AskAgent]  需求理解与多轮澄清（最多3轮）
  ↓ 需求已明确
[PlanAgent] 技术路线规划 + 数据流校验
  ↓ 生成执行计划
[CraftAgent] 按计划调用工具执行，失败可重试
  ↓
分析报告输出 → reports/ 目录
```

### 3.3 FC 循环工作原理

```
用户输入 → [LLM 决策] → 有工具调用?
                            ├─ 是 → 执行工具 → 结果回填消息 → 继续循环（最多10轮）
                            └─ 否 → 直接返回回复
```

### 3.4 校准弃权机制

工具执行后由 `ResultEvaluator` 评估结果可靠性，若可信度低于 0.7 则在结果中附加 `⚠️ [校准弃权]` 警告，建议专家介入。

## 四、工具层（37 个生信工具）

| 分类 | 工具文件 | 包含工具 |
|------|----------|---------|
| 序列处理 | `seqkit_tool.py` | seqkit（统计/过滤/搜索等5个） |
| 质量控制 | `qc_tool.py` | fastp / fastqc / multiqc / cutadapt |
| 序列组装 | `assembly_tool.py` | SPAdes / MEGAHIT |
| 序列比对 | `alignment_tool.py` | minimap2 / samtools / BLASTn / DIAMOND |
| 进化分析 | `evolution_tool.py` | MAFFT / trimAl / IQ-TREE / CodeML |
| 物种分类 | `taxonomy_tool.py` | Kraken2 |
| 信息检索 | `search_tool.py` `pubmed_tool.py` `web_fetch_tool.py` | Web搜索 / PubMed / 网页抓取 |
| 知识库 | `knowledge_tool.py` | VITALdb + 本地知识库检索 |
| 可视化 | `viz_tool.py` | Circos 等 |

## 五、工作流引擎

`workflow/` 目录提供三套预设 YAML 工作流：

| 工作流文件 | 用途 |
|-----------|------|
| `virus_analysis.yaml` | 病毒全流程分析 |
| `quality_check.yaml` | 质控报告 |
| `phylogenetic_analysis.yaml` | 系统发育分析 |

另有 `templates/` 目录存放 Nextflow 模板（`.nf` 格式）支持更复杂的流水线。

## 六、LLM 配置

支持多 Provider 自动检测和切换：

- **默认**：MiniMax 远程 API（`MiniMax-M2.7`）
- **备用**：本地 Ollama（如 `qwen3:4b`）
- **其他**：心流、硅基流动等（`api_providers.yaml` 配置）

## 七、使用方法

### 7.1 环境安装

```bash
cd /home/bgi-chao/lihuyang/flu_agent

# Conda 环境安装
conda env create -f environment.yml
conda activate bioagent

# 或 pip 安装
pip install -r requirements.txt
```

### 7.2 三种启动模式

```bash
# 1. CLI 命令行交互模式（最简单）
python run.py --mode cli

# 2. Web UI 模式（Gradio 旧版界面）
python run.py --mode web --port 7861

# 3. API 服务模式（配合 Next.js 前端）
python run.py --mode api --port 8000

# 启动 Next.js 前端（API 模式配套）
cd frontend && npm run dev
```

### 7.3 管理命令

```bash
# 检查生信工具可用性
python manage.py check-tools

# 检查 LLM 连接
python manage.py check-llm

# 初始化数据库
python manage.py init-db

# 查看会话列表
python manage.py sessions

# 清理缓存
python manage.py clean-cache
```

### 7.4 主要配置文件

| 文件 | 用途 |
|------|------|
| `config.yaml` | 主配置：LLM、工具路径、Agent参数、Docker、任务队列 |
| `api_providers.yaml` | 多 LLM Provider 配置 |
| `environment.yml` | Conda 环境依赖 |

### 7.5 CLI 对话示例

```
用户> 对 data/test_fasta.fa 文件做序列统计
FluAgent> [自动调用 seqkit_stats 工具，返回统计结果]

用户> 分析这批序列的系统发育关系
FluAgent> [自动规划：MAFFT比对 → trimAl修剪 → IQ-TREE建树 → 输出报告]

用户> quit
是否保存本次会话内容到 reports 目录? (y/n): y
✅ 已保存到: reports/analysis_20260408_223900.md
```

## 八、关键特性汇总

| 特性 | 说明 |
|------|------|
| **多模式 Agent** | FC 循环（简单任务）+ 三角色 Ask-Plan-Craft（复杂任务） |
| **多 LLM 支持** | 本地 Ollama / 多个远程 API 自动切换 |
| **校准弃权** | 结果可信度评估，不确定时主动提示专家介入 |
| **会话持久化** | SQLite 存储对话历史，支持多用户隔离 |
| **Docker 沙箱** | 可选容器化执行，隔离工具运行环境 |
| **异步任务队列** | Celery + Redis，支持长时间任务异步提交 |
| **模糊工具纠错** | LLM 调用工具名错误时自动模糊匹配并纠正 |
